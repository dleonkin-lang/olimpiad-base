# Prompt для первого запуска в Codex: только анализ, без изменений

Ты работаешь с репозиторием статической базы олимпиадных задач 5–7 класса.

Сделай только разведочный проход. Ничего не меняй в файлах, не форматируй, не пересобирай, не коммить и не удаляй. Не запускай команды, которые могут модифицировать рабочее дерево. Если нужна проверка, используй только read-only команды.

Твоя задача:

1. Посмотри структуру репозитория.
2. Прочитай `README.md`, `AGENTS.md`, `project_docs/PROJECT_STRUCTURE.md` и этот файл.
3. Определи, какие файлы являются входными точками сайта.
4. Проверь, что GitHub Pages должен публиковать папку `docs/`.
5. Осмотри `docs/index.html` и `docs/admin.html`: найди, где расположен `const DATA`, где задан `CAN_EDIT`, как устроены основные функции интерфейса.
6. Проверь, что публичная версия read-only (`CAN_EDIT = false`), а рабочая версия editable (`CAN_EDIT = true`).
7. Проверь, что в данных 237 задач и 51 источник. Можно использовать `python3 tools/verify_static_bundle.py`, если он не меняет файлы.
8. Объясни, как связаны HTML и JSON: данные встроены в HTML, а JSON лежит как бэкап/источник для будущей пересборки.
9. Найди потенциальные риски для дальнейшей работы: рассинхронизация HTML и JSON, случайная потеря задач, изменение рубрик, публикация лишних материалов, различие admin/read-only.
10. В конце дай короткий отчёт:
    - структура проекта;
    - точки входа;
    - как работает интерфейс;
    - какие инварианты нельзя нарушать;
    - какие вопросы нужно задать перед изменениями.

Ограничение: на этом первом проходе не вносить никаких изменений.
