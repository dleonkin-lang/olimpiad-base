# Рабочий процесс

## Две ветки работы

- Ветка данных: импорт новых задач, источников, решений, ответов, картинок, переводов и рубрикации задач.
- Ветка интерфейса: внешний вид, кнопки, фильтры, конструктор листочков, полноэкранный режим, read-only поведение.

При слиянии данные берутся из ветки данных, интерфейс — из ветки интерфейса.

## Перед изменениями

1. Сделать архив текущей версии как точку отката.
2. Проверить количество задач и источников.
3. Проверить `CAN_EDIT` в обеих HTML-версиях.

## После изменений

1. Проверить `python3 tools/verify_static_bundle.py`.
2. Обновить JSON-бэкап, если изменились данные.
3. Выдать полный комплект артефактов: рабочий HTML, публичный HTML, JSON базы задач, полный JSON-бэкап, каталог источников, архив версии.
